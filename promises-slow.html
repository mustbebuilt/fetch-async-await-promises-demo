      Explanation

      This example uses fetch which returns a promise.

      A promise is an object that represents the eventual completion or failure of an asynchronous operation.

      A promise has three states: pending, fulfilled, and rejected.

      A promise can be in one of these states at any given time.

      Asyncronous

      Fetch calls are asyncronous.This is because the promises are none blocking and allow for other tasks to run.

      This is shown in the demo as the API has been deliberately slowly to take 5 seconds to respond. You can see that this does not stop the setInterval from running which is been used to count the number of seconds that the API has been called.

      The value of a promise can be retrieved using the .then and .catch methods.

      For example, if you try to access the value of a promise in the .then method, it will not return anything until the promise is fulfilled.

      When a promise is fulfilled, it will return the value of the promise but not the data from the server.

      The promise is passed to a second .then method as a callback function which is then able to access the data from the server most commonly via the .json method.

      Finally, once the data is received, it can be added to the DOM to be displayed.
